Key Features

  • Per-Application Control: Rules are based on the application, not IP addresses
  • Smart Decisions: Choose "once" for temporary access or "always" to remember
  • Clear Information: See exactly which app is connecting and where
  • Automatic Caching: "Always" rules are applied automatically for future connections
  • Connection Logging: All connection attempts are logged for audit

How It Works

  1. Monitor: The daemon monitors all outgoing network connections
  2. Identify: Each connection is traced back to its application
  3. Check Rules: Existing rules are applied automatically
  4. Ask User: If no rule exists, a popup dialog appears
  5. Apply Decision: Your choice is stored and applied immediately

Use Cases

Web Browser

When Firefox connects to a website, choose "Always allow" so it can access any site without repeated prompts.

Unknown Application

When an unfamiliar application tries to connect, review the details and choose "This time only" if you're unsure, or "Deny" to block it.

Testing

For applications you're testing, use "This time only" to allow connections without creating permanent rules.
